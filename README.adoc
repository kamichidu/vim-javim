= javim: Vim Script JVM (Vim 8)

`javim` is a lightweight, pure Vim Script implementation of a Java Virtual Machine (JVM). It runs on Vim 8+ (excluding Neovim) and executes Java `.class` bytecode files directly inside your Vim session.

== Features

* **Pure Vim Script Implementation:** Built using native Vim 8 data structures, including `Blob` for high-performance class file decoding and `Dictionary` objects for the JVM heap and stack frames.
* **Vim-Native Standard Library (`rt/`):** Key classes like `java.lang.Object`, `java.lang.String`, `java.lang.System`, and `java.io.PrintStream` are implemented natively in Vim Script, eliminating the need to bundle a bulky external `rt.jar`.
* **Dynamic Method Dispatch:** Supports inheritance and interfaces with runtime resolution (`invokevirtual`, `invokespecial`, `invokestatic`).
* **Vim 8 Exclusive Compatibility:** Leverages standard Vim 8 binary blobs and test assertions.

== Usage

To compile and run your Java classes inside Vim:

. Write your Java class (e.g., `HelloWorld.java`):
+
[source,java]
----
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
----

. Compile the class to bytecode using the standard Java compiler:
+
[source,bash]
----
javac HelloWorld.java
----

. Open Vim and execute the class using the `:JavimRun` command:
+
[source,vim]
----
" Run from current directory (default)
:JavimRun HelloWorld

" Run with a custom classpath (colon ':' on macOS/Linux, semicolon ';' on Windows)
:JavimRun -cp bin:lib HelloWorld
----

== Configuration

g:javim_cache_dir::
  The root directory used for extracting and caching JAR files. If unset, defaults to `~/.cache/javim`.

== Limitations & Compatibility

`javim` is a lightweight, pure Vim Script interpreter designed strictly around the Java Virtual Machine Specification, Java SE 8 Edition (JVMS SE 8).

=== Requirements

External `unzip` command::
  A standard `unzip` utility must be installed and accessible in your system `PATH`. This is a mandatory requirement to support `jar file classpath entry` classloading.

=== Supported Classpath Features

The classloader parses custom classpath configurations passed via `-cp` or `-classpath` according to the following support matrix:

Directory (`directory`):: Supported. Looks up `.class` files sequentially under specified directory structures (separated by `:` on macOS/Linux, `;` on Windows).
JAR file classpath entry (`jar file classpath entry`):: Supported via extraction cache. Direct `.jar` file paths can be specified. `javim` extracts the JAR archive under your configured cache directory (using mtime-based hashing to handle updates) using the external `unzip` command, and loads classes from the extracted directory.
Wildcard (`wildcard`):: Supported. Wildcard patterns (e.g. `lib/*`) are expanded into matching file paths using Vim's built-in `glob()` function at startup. Note that unlike standard JREs (which strictly match only `.jar`/`.JAR` files under the specified directory), `javim` delegates parsing to Vim's native globbing engine and then filters the results to keep only valid directories and `.jar` files.

=== Unsupported Packaging Features

Project Jigsaw (Java 9+ Module System):: Explicitly **unsupported**. Modular directory structure and module-path command-line flags (`--module-path` or `-p`) are completely ignored.
JAR Manifest (`Class-Path` header):: Explicitly **unsupported**. Automatic resolution and dependent loading of paths listed in the `Class-Path` header inside a JAR's `META-INF/MANIFEST.MF` is not supported. Only classes contained directly inside the specified JAR are loaded.

== Troubleshooting

If you encounter class-loading or runtime errors:

ClassNotFoundException:: Ensure that the directory containing your compiled `.class` files is included in the JVM classpath (by default, it searches the current directory).
NullPointerException:: Occurs when attempting to perform an operation on a null object reference.
Unsupported opcode:: A specific JVM bytecode instruction is not yet implemented in `instructions.vim`. Please report this with the opcode hex representation.
