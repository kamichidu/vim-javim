= javim: Pure Vim Script Java Virtual Machine (JVM)

`javim` is a minimal implementation of a Java Virtual Machine (JVM) written in pure Vim Script. It runs on Vim 8+ (excluding Neovim) and executes Java `.class` bytecode files inside Vim without requiring an external Java Runtime Environment (JRE) at runtime.

== Features

* *Run Java with Only Vim:* Execute Java bytecode directly inside Vim without needing an external JVM or JRE at runtime.
* *Vim-Native Standard Library (`rt/`):* Core standard classes like `java.lang.Object`, `java.lang.String`, `java.lang.System`, and `java.io.PrintStream` are provided as built-in runtime classes in Vim Script, bypassing the need for a bulky compiled `rt.jar` (see link:docs/API.md[API.md] for the complete support matrix).
* *Pure Vim Script Implementation:* Leverages standard Vim 8 binary `Blob` data structures for high-performance class file decoding, and `Dictionary` objects for the JVM heap and stack frames.
* *Dynamic Method Dispatch:* Full runtime method resolution support for inheritance and interfaces (`invokevirtual`, `invokespecial`, `invokestatic`).
* *Vim 8 Exclusive:* Tailored to standard Vim 8 specifications with robust binary blob parsing and built-in test assertions.

== Installation

You can install `javim` using your favorite Vim plugin manager.

Using *vim-plug*:
[source,vim]
----
Plug 'kamichidu/vim-javim'
----

Using Vim's native package manager:
[source,bash]
----
mkdir -p ~/.vim/pack/plugins/start
cd ~/.vim/pack/plugins/start
git clone https://github.com/kamichidu/vim-javim.git
----

=== Requirements

External `unzip` command::
  Only required if you want to load classes from `.jar` files in your classpath. Otherwise, a standard Vim 8+ installation is all you need to execute compiled `.class` files.

== Usage

To compile and run your Java classes inside Vim:

. Write your Java class (e.g., `HelloWorld.java`):
+
[source,java]
----
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
----

. Compile the class to Java bytecode using your local compiler:
+
[source,bash]
----
javac HelloWorld.java
----

. Open Vim and execute the class using the `:JavimRun` command:
+
[source,vim]
----
" Run from current directory (default)
:JavimRun HelloWorld

" Run with a custom classpath (colon ':' on macOS/Linux, semicolon ';' on Windows)
:JavimRun -cp bin:lib HelloWorld
----

== Troubleshooting

ClassNotFoundException:: Ensure that the directory containing your compiled `.class` files is included in the JVM classpath (by default, it searches the current directory).
NullPointerException:: Occurs when attempting to perform an operation on a null object reference.
Unsupported opcode:: A specific JVM bytecode instruction is not yet implemented in `instructions.vim`. Please report this with the opcode hex representation.

== Configuration

g:javim_cache_dir::
  The root directory used for extracting and caching JAR files. If unset, defaults to `~/.cache/javim`.

== Internal Architecture & Implementation

`javim` interprets compiled class files through a modular design written entirely in Vim Script.

=== Module Components

Decoder (`autoload/javim/classfile.vim`)::
  Reads Java bytecode `.class` files as a binary `Blob` using Vim's native `readfile(..., 'B')` and parses constant pools, interfaces, fields, methods, and bytecode attributes.
Interpreter (`autoload/javim/interpreter.vim`)::
  Manages JVM stack frames, heap allocation, static fields, and the main execution loop.
Instruction Set (`autoload/javim/instructions.vim`)::
  Implements standard JVM opcodes (arithmetic, object initialization, loops, branch jumps, static and dynamic method dispatch). For supported instructions, see link:docs/OPCODES.md[OPCODES.md].
Vim-Native Runtime (`autoload/javim/rt/`)::
  Bypasses the need for real compiled standard libraries (`rt.jar`). Standard runtime classes are provided as built-in runtime classes in Vim Script (see link:docs/API.md[API.md]).

=== Classpath Resolution Details

The classloader parses custom classpath configurations passed via `-cp` or `-classpath` according to the following support matrix:

Directory (`directory`):: Supported. Looks up `.class` files sequentially under specified directory structures (separated by `:` on macOS/Linux, `;` on Windows).
JAR file classpath entry (`jar file`):: Supported via extraction cache. Direct `.jar` file paths can be specified. `javim` extracts the JAR archive under your configured cache directory (using mtime-based hashing to handle updates) using the external `unzip` command, and loads classes from the extracted directory.
Wildcard (`wildcard`):: Supported. Wildcard patterns (e.g., `lib/*`) are expanded into matching file paths using Vim's built-in `glob()` function at startup. Note that unlike standard JREs (which strictly match only `.jar`/`.JAR` files under the specified directory), `javim` delegates parsing to Vim's native globbing engine and then filters the results to keep only valid directories and `.jar` files.

== Limitations & Compatibility

Besides core JVM spec support (detailed in link:docs/JVMS.md[JVMS.md]), packaging and loading limitations include:

=== Unsupported Features

Project Jigsaw (Java 9+ Module System):: Explicitly *unsupported*. Modular directory structure and module-path command-line flags (`--module-path` or `-p`) are completely ignored.
JAR Manifest (`Class-Path` header):: Explicitly *unsupported*. Automatic resolution and dependent loading of paths listed in the `Class-Path` header inside a JAR's `META-INF/MANIFEST.MF` is not supported. Only classes contained directly inside the specified JAR are loaded.
